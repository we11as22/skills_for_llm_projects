# Session Checkpoint Template

Use this after ~10 turns of code changes to stay aligned with the user.

```
SESSION CHECKPOINT
──────────────────
Goal stated:    [what the user asked for at the start]
Completed:
  - [file/change 1]
  - [file/change 2]
  - ...
Pending:
  - [item not yet done]
  - ...
Open questions: [anything unclear or unresolved]
Next step:      [what I plan to do next]

Does this match your understanding?
```
